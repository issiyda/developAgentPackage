# 記事情報自動取得スキル

このスキルは、指定されたURLから最新の記事情報を自動的に取得し、構造化されたデータとして提供します。

## 使用方法

ユーザーがURLを提供した際、以下の情報を自動的に取得します：

1. **記事のタイトル**
2. **記事の本文内容**
3. **公開日時**
4. **著者情報**
5. **カテゴリー・タグ**
6. **画像URL**
7. **メタデータ**

## 実行手順

### 1. URLの確認
ユーザーから提供されたURLが有効であることを確認します。

### 2. 記事情報の取得
WebFetchツールを使用して、指定されたURLから情報を取得します。

### 3. 情報の構造化
取得した情報を以下の形式で構造化します：

```
## 記事情報

### 基本情報
- **タイトル**: [記事のタイトル]
- **URL**: [記事のURL]
- **取得日時**: [現在の日時]

### コンテンツ
- **本文**: [記事の本文内容（要約または全文）]

### メタデータ
- **公開日**: [公開日時]
- **著者**: [著者名]
- **カテゴリー**: [カテゴリー情報]
- **タグ**: [タグ情報]

### メディア
- **画像**: [画像URLのリスト]
```

### 4. 結果の提示
取得した情報を見やすく整形してユーザーに提示します。

## 使用例

**ユーザー**: https://example.com/article/latest-news の記事情報を取得して

**アシスタント**: WebFetchツールを使用して記事情報を取得し、構造化されたデータとして提示します。

## 自動実行のトリガー

以下のようなリクエストで自動的に実行されます：

- "このURLの記事情報を取得して"
- "https://... の最新情報を確認して"
- "記事の内容を取得してください"
- "このページの情報を抽出して"

## 注意事項

- URLが有効でない場合やアクセスできない場合はエラーメッセージを表示します
- サイトによってはアクセス制限がある場合があります
- 取得できる情報はサイトの構造によって異なる場合があります

## 実装詳細

WebFetchツールを活用して以下の処理を実行します：

```
1. URL検証
2. WebFetch実行（以下のプロンプトを使用）:
   "この記事の以下の情報を抽出してください：
    - タイトル
    - 本文内容
    - 公開日時
    - 著者
    - カテゴリーやタグ
    - 画像URL
    すべて構造化された形式で提供してください。"
3. 結果の整形と提示
```

このスキルにより、記事情報が更新されても常に最新の情報を自動的に取得できます。
