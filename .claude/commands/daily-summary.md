---
description: 今日の作業内容を要約して明日やるべきことをリストアップする
---

今日の作業を振り返り、明日への準備を行います。以下の手順で実行してください:

## 1. 今日の作業内容の収集

以下の情報を並列で収集してください:
- `git log --since="midnight" --pretty=format:"%h - %s (%an, %ar)" --no-merges` で今日のコミット履歴を取得
- `git diff --stat $(git log --since="midnight" --format="%H" | tail -1)^ HEAD` で今日の変更ファイル統計を取得
- `git status` で現在の未コミット変更を確認

## 2. 作業内容の分析

収集した情報から以下を分析してください:
- 実装した機能やバグ修正の一覧
- 変更したファイルの種類と範囲
- コミットメッセージから読み取れる作業の流れ
- 現在進行中（未コミット）の作業

## 3. TODOコメントの検索

コードベース内の残タスクを確認してください:
- `TODO`, `FIXME`, `HACK`, `XXX` などのコメントを検索
- 特に今日変更したファイル周辺のTODOを優先的にリストアップ

## 4. 要約レポートの作成

以下の形式で日本語のレポートを作成してください:

```markdown
# 今日の作業サマリー (YYYY-MM-DD)

## 完了した作業
- [コミット履歴から抽出した作業内容をリスト形式で]
- [各項目には変更ファイル数や主要な変更点を含める]

## 変更ファイル統計
- 変更されたファイル数: X件
- 追加行数: +XXX
- 削除行数: -XXX
- 主な変更領域: [フロントエンド/バックエンド/設定ファイルなど]

## 現在進行中の作業
- [未コミットの変更があれば記載]
- [作業の進捗状況を簡潔に]

## 明日やるべきこと

### 優先度: 高
1. [今日の作業の続きや未完了タスク]
2. [コード内のTODOコメントから抽出]
3. [テストやドキュメント更新など]

### 優先度: 中
1. [リファクタリングや改善項目]
2. [FIXMEコメントの対応]

### その他の検討事項
- [長期的なタスクや技術的負債]
- [パフォーマンス改善の余地]

## 注意事項・メモ
- [気づいた問題点や改善のアイデア]
- [次回作業時に注意すべきポイント]
```

## 5. レポートの保存（オプション）

必要に応じて、AskUserQuestionツールを使用してユーザーに以下を確認してください:
- レポートをファイルとして保存するか（例: `docs/daily-reports/YYYY-MM-DD.md`）
- レポートを画面表示のみにするか

ユーザーが保存を希望する場合は、適切なディレクトリに保存してください。

## 6. 追加の提案

分析結果に基づいて、以下のような追加アクションを提案してください:
- 未コミットの変更が多い場合: コミットを推奨
- TODOが多数ある場合: GitHub Issuesへの登録を提案
- テストカバレッジが不足している場合: テスト追加を提案

---

**注意事項:**
- コミット履歴がない日でも、現在の状態を確認してレポートを作成してください
- 機密情報（APIキー、パスワードなど）がコミットメッセージに含まれていないか確認してください
- 複数人で開発している場合は、自分のコミットのみをフィルタリングすることも検討してください
